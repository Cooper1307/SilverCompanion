# 上海杉达学院《人工智能通识教育课程》项目实践报告提交规范

根据课程要求及报告模板，请各项目小组在最终提交时严格遵守以下排版及格式规范。

## 1. 页面与排版设置 (Layout)

* **纸张大小**：A4 (210mm × 297mm)
* **页边距**：
  * 上页边距：2.5 厘米
  * 下页边距：2.5 厘米
  * 左页边距：3.1 厘米
  * 右页边距：3.1 厘米
* **行间距**：多倍行距 1.3 倍

## 2. 字体与字号规范 (Fonts & Size)

| 层次 | 字体 | 字号 | 备注 |
| :--- | :--- | :--- | :--- |
| **主标题 (大标题)** | 黑体 (SimHei) | 三号 (16pt) | 居中，加粗 |
| **一级标题** | 仿宋 (FangSong) | 小三 (15pt) | 靠左，加粗建议 |
| **正文内容** | 仿宋 (FangSong) | 小四 (12pt) | 两端对齐 |
| **表格标题/内容** | 仿宋/黑体 | 小四或五号 | 视内容紧凑程度而定 |

## 3. 提交成果形式 (Deliverables)

1. **正式报告**：
    * 必需提交 **Word 文档 (.docx)** 格式。
    * 排版必须完全符合上述第 1、2 条要求。
2. **演示原型 (可选但推荐)**：
    * 鼓励开发可演示的智能体原型（如：对话机器人、简易应用等）。
    * 需提供访问链接或二维码，并确保在评审期间可公开访问。

## 4. 内容完整性要求 (Checklist)

提交前请自查是否包含以下核心章节：
* [ ] **项目基本信息**：含项目名称、负责人及详细团队分工（含技术、协作、创新等维度评价）。
* [ ] **项目概述**：含挑战背景、解决方案及服务对象。
* [ ] **技术方案设计**：含数据准备、模型/平台选择及业务/智能体工作流图。
* [ ] **开发过程与关键决策**：含开发拆解、Prompt 设计示例及技术/非技术难题记录。
* [ ] **成效评估**：含部署访问方式、功能截图及技术指标/用户反馈。
* [ ] **项目反思与迁移**：含跨场景迁移思考表。
* [ ] **伦理合规性自查**：包含隐私保护、算法公平性、适老化设计等自查项。

## 5. 特别注意事项

* **团队评分**：成员分工评价需涵盖技术掌握、团队协作、创新思维、社会责任感、伦理意识五个维度（1-5分）。
* **跨场景迁移**：必须包含一个全新的社会问题迁移构想（如面向残障人士或留守儿童）。
* **合规性说明**：自查表中的“具体实现方式说明”应针对本项目实际采取的措施进行具体描述。
